Swaps: Do onchain to offchain and vice-versa swaps

Loop out and Loop in funcionality on LNbits

GET /swap/api/v1/out
Body (application/json)
Returns 200 OK (application/json)
JSON list of swap outs
Curl example
curl -X GET {{ root_url }}/swap/api/v1/out -H "X-Api-Key: {{ user.wallets[0].inkey }}" POST /swap/api/v1/out
Headers
{"X-Api-Key": <string>, "Content-type": "application/json"}
Body (application/json) - "wallet" is a YOUR wallet ID
{"wallet": <string>, "onchainwallet": <Optional string>, "onchainaddress": <string>,"amount": <integer>,"recurrent": <Optional boolean> ,"fee": <integer>}
Returns 201 CREATED (application/json)
{"id": <string>, "wallet": <string>, "onchainwallet": <string>, "onchainaddress": <string>, "amount": <integer>, "recurrent": <boolean>, "fee": <integer>}
Curl example
curl -X POST {{ root_url }}/swap/api/v1/out -d '{"wallet": "{{ user.wallets[0].id }}", "onchainwallet": <Optional string>, "onchainaddress": <string>,"amount": <integer>,"recurrent": <Optional boolean> ,"fee": <integer>}' -H "X-Api-Key: {{ user.wallets[0].adminkey }}" -H "Content-type: application/json" GET /swap/api/v1/out
Body (application/json)
Returns 200 OK (application/json)
JSON list of recurrent swaps
Curl example
curl -X GET {{ root_url }}/swap/api/v1/recurrent -H "X-Api-Key: {{ user.wallets[0].inkey }}" POST /swap/api/v1/out
Headers
{"X-Api-Key": <string>}
Curl example
curl -X POST {{ root_url }}/swap/api/v1/out -d '{"wallet": <string>, "onchainwallet": < Optional string>,"onchainaddress": <string>, "threshold": <integer>, "fee": <integer>}' -H "X-Api-Key: {{ user.wallets[0].adminkey }}" -H "Content-type: application/json" DELETE /swap/api/v1/recurrent/<recurrent_id>
Headers
{"X-Api-Key": <string>}
Curl example
curl -X DELETE {{ root_url }}/swap/api/v1/recurrent/<recurrent_id> -H "X-Api-Key: {{ user.wallets[0].adminkey }}"
